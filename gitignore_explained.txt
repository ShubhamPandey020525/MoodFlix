MoodFlix - Gitignore Reference (Personal Notes)

I have configured the .gitignore file to maintain a clean and professional repository. Below is a detailed breakdown of the ignored files for future reference:

1. Sensitive Information (.env, .env.*)
   - Contains TMDB API keys and other private environment variables.
   - Crucial: These must never be public to ensure security and prevent unauthorized access.

2. Data & Datasets (*.csv)
   - Large movie datasets (e.g., movies_cleaned.csv) are ignored to keep the repository size manageable.
   - Prevents issues with GitHub's file size limits.

3. AI Models (*.pkl, *.h5)
   - Trained AI models are binary files that consume significant storage space.
   - Ignored to optimize repository performance and follow best practices for machine learning projects.

4. Dependencies (node_modules)
   - Standard industry practice: Local libraries can be re-installed using 'npm install', so they are not tracked.

5. Build Output (dist, dist-ssr)
   - Production-ready assets generated by the build command.
   - Only source code is tracked to maintain a clean development history.

6. Legacy Configurations (.lovable/)
   - Configuration files from previous development tools that are no longer required for the current setup.

7. System & Editor Files (.vscode, .DS_Store)
   - Personal editor settings and operating system temporary files are ignored to maintain a professional workspace across different environments.

Summary: The repository is optimized to protect sensitive API credentials and maintain a lightweight, efficient codebase.
